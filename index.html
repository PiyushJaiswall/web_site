Namespace Management

  • Classes written so far all belong to a single name space: a unique name has to be chosen for each class to avoid name collision
  • Java provides a mechanism for partitioning the class namespace into more manageable chunks. This mechanism is a package i.e In Java, a package is a namespace that organizes a set of related classes and interfaces. It provides a way to group related classes and interfaces, making them easier to manage and maintain .

Packages Help in:

  1. Avoiding naming conflicts: By organizing classes and interfaces in packages, naming conflicts can be avoided.
  2. Access control: Packages allow for access control. You can specify which classes can access a particular package or class.
  3. Code reuse: Packages allow for code reuse. You can reuse classes and interfaces across different projects.
  4. Modularity : Packages provide a way to modularize code. By breaking code into smaller, more manageable packages, you can make it easier to understand and maintain.
  5. Separation of concerns: Packages allow for separation of concerns. You can separate your code into different packages based on functionality, making it easier to understand and modify.

Creating Packages In Java

Creating a package in Java involves organizing related classes and interfaces into a single namespace to improve code organization and reduce naming conflicts.

Here are the steps involved in creating a package:

  1. Choose a package name. It is recommended to use a reverse domain name format for your package name to ensure uniqueness and prevent naming conflicts. For example, if your domain name is "example.com", you can use "com.example" as your package name.
  2. Create a directory structure that corresponds to your package name. For example, if your package name is "com.example", you need to create a directory called "com" inside your project directory. Inside the "com" directory, create another directory called "example". Your directory structure should look like this: 
    project/
  └── com/
      └── example/
  3. Create your Java classes and interfaces. Save each class or interface in a separate file inside the appropriate directory. For example, if you have a class called "MyClass" in the "com.example" package, you would save it in a file called "MyClass.java" inside the "com/example" directory.
  4. Add a package declaration at the beginning of your Java file to indicate that a class or interface belongs to a particular package. The package declaration should match the directory structure that you created earlier. For example, if you have a class called "MyClass" in the "com.example" package, you would include the following package declaration at the beginning of your Java file: 
    package com.example;
  
  public class MyClass {
    // class code here
  }
  5. Compile your Java code. To compile all the Java files in your package, navigate to the directory that contains the top-level package directory (in our example, the "com" directory), and use the "javac" command to compile all the files in the package: 
    javac com/example/*.java
    This command compiles all the Java files in the "com.example" package and generates class files for each file in the same directory.
  6. Use your package. Once you have compiled your package, you can use it in your Java code by importing the package using the "import" statement. For example, if you have a class called "MyClass" in the "com.example" package, you would import it in another class as follows: 
    import com.example.MyClass;
  
  public class MyOtherClass {
    public static void main(String[] args){
      MyClass myObject = new MyClass();
      // use myObject here
    }
  }

Package Hierarchy

Package hierarchy in Java is the way of organizing related packages into a tree-like structure where each package can contain sub-packages, and each sub-package can contain further sub-packages, creating a hierarchical structure of packages.

The top-level package in Java is the default package, which contains classes and interfaces with no package declaration. All other packages are organized hierarchically under this default package.


For example, consider a project that involves different packages like: 

                com.example
                ├── app
                │   ├── controller
                │   ├── model
                │   └── view
                └── util
                ├── encryption
                └── logging
In this package hierarchy, the root package is com.example, which contains two sub-packages app and util. The app package has three sub-packages controller, model, and view. Similarly, the util package contains two sub-packages encryption and logging.
The benefit of organizing packages in a hierarchical structure is that it makes it easier to manage large codebases. By grouping related packages together, you can quickly locate and modify code. It also makes it easier to reuse code across multiple projects, as you can simply import the packages you need and use them in your code.

For instance, if you want to use the logging package in another project, you can import it by using the statement:
import com.example.util.logging;

This will give you access to all the classes and interfaces inside the logging package and its sub-packages.

In addition, package hierarchy also helps in avoiding naming conflicts between packages. When packages are organized hierarchically, each package has a unique namespace, which helps in avoiding naming conflicts between classes and interfaces.

Package Finding In Java

Package finding in Java is the process of locating and importing the necessary packages to use classes and interfaces in your code. When you want to use a class or interface that is defined in another package, you need to import that package into your code.
There are two ways to import a package in Java:

  1. Using the fully qualified name of the class or interface:
    2. You can use the fully qualified name of the class or interface, which includes the package name, to use it in your code. For example, if you want to use the ArrayList class from the java.util package, you can use the following code:

    java.util.ArrayList myList = new java.util.ArrayList();


    In this case, you are using the fully qualified name of the ArrayList class, which includes the package name java.util. This is not an ideal approach because it can make your code harder to read and maintain, especially if you have to use a lot of classes and interfaces from different packages.

  2. Using the import statement:
    3. The import statement is used to import a package or specific classes and interfaces from a package. This allows you to use the classes and interfaces in your code without using their fully qualified names.
    For example, to import the ArrayList class from the java.util package, you can use the following code:
                      import java.util.ArrayList;


                  ...
                  
                  
                  
                  ArrayList myList = new ArrayList();


    In this case, you are using the import statement to import the ArrayList class from the java.util package. This makes your code more readable and maintainable because you don't have to use the fully qualified name of the class every time you want to use it.
    To import all the classes and interfaces in a package, you can use the wildcard character *. For example, to import all the classes and interfaces from the java.util package, you can use the following code:

    import java.util.*;


    This imports all the classes and interfaces from the java.util package, which means you can use them in your code without using their fully qualified names.

Importing Packages In Java

In Java, packages are used to organize related classes and interfaces, making it easier to manage and maintain large codebases. To use a class or interface from a different package, you need to import the necessary packages into your code. Importing packages in Java can be done in several ways.


  1. Importing a Single Class
    2. You can import a single class from a package using the following syntax:

    import package.name.ClassName;


    For example, if you want to use the ArrayList class from the java.util package, you can import it using the following statement:

    import java.util.ArrayList;


    After importing the class, you can create an instance of it using the following code:

    ArrayList myList = new ArrayList<>();


  2. Importing Multiple Classes from the Same Package

    3. If you need to use multiple classes from the same package, you can import them all at once using the following syntax:

     import package.name.*;


    For example, if you want to use the ArrayList and LinkedList classes from the java.util package, you can import them using the following statement:

    import java.util.*;


    After importing the classes, you can create instances of them using the following code:

    ArrayList myList = new ArrayList<>();

    LinkedList myLinkedList = new LinkedList<>();

  3. Importing a Static Member

    4. You can import a static member of a class using the following syntax:

    import static package.name.ClassName.staticMember;


    For example, if you want to use the PI constant from the java.lang.Math class, you can import it using the following statement:

     import static java.lang.Math.PI;


    After importing the constant, you can use it in your code directly:


    double radius = 10.0;
    double circumference = 2 * PI * radius;

  4. Importing All Static Members

    5. You can import all the static members of a class using the following syntax:

    import static package.name.ClassName.*;


    For example, if you want to use all the static members of the Math class from the java.lang package, you can import them using the following statement:

       import static java.lang.Math.*;


    After importing the static members, you can use them in your code directly:

    double radius = 10.0;
    double circumference = 2 * PI * radius;
    double sineValue = sin(PI/2);

Access Control Of Packages

Access control in Java is the process of controlling the visibility and accessibility of members of a class. It allows you to specify who can access the members of a class and from where they can be accessed. This is important in object-oriented programming because it helps to encapsulate the implementation details of a class and protect it from unwanted modification or access.


    In Java, there are four access control modifiers that can be used to control the accessibility of class members: private, default (package-private), protected, and public.

  1. private:
    2. A private member is only accessible within the same class. It cannot be accessed from outside the class, even from its subclasses.

  2. default:
    3. A member with no access modifier (also known as package-private) is only accessible within the same package. It cannot be accessed from outside the package, even from its subclasses.

  3. protected:
    4. A protected member is accessible within the same package and also from its subclasses (even if they are in a different package).

  4. public:
    5. A public member is accessible from anywhere in the program.

    The table below summarizes the access level of class members based on their access modifiers:
    Access Modifier Within class Within package Subclass in different package Anywhere
    Private Yes No No No
    Default Yes Yes No No
    Protected Yes Yes Yes No
    Public Yes Yes Yes Yes


    Here are some examples of how access control works in Java:
    // Example 1: private access modifier
                        public class MyClass {
                    
                        private int privateField;
                    
                        private void privateMethod() {
                            // code here
                        }
                    
                    }
                                                        
                        // The privateField and privateMethod cannot be accessed from outside the MyClas                                      
                        // class, even from its subclasses.




    // Example 2: default access modifier 
                        class MyClass {
                        int defaultField;                                        
                        void defaultMethod() {                                                
                            // code here                                              
                        }                                              
                    }                                            
                    
                    // The defaultField and defaultMethod can be accessed within the same package,                                                
                    // but not from outside the package.




    // Example 3: protected access modifier 
                        public class MyClass {                                                    
                        protected int protectedField;                                                   
                        protected void protectedMethod() {                                                        
                            // code here                                                        
                        }                                                        
                    }                                                       
                    // The protectedField and protectedMethod can be accessed within the same package                     
                    // and also from its subclasses, even if they are in a different package.




    // Example 4: public access modifier 
                        public class MyClass {
                        public int publicField;
                        
                        public void publicMethod() {
                            // code here
                        }
                    }
                    // The publicField and publicMethod can be accessed from anywhere in the program.

Built In Packages In Java

Java provides a large number of built-in packages that contain classes and interfaces for performing a wide range of tasks, from basic input/output operations to advanced networking and database connectivity. Here are some of the most commonly used built-in packages in Java:


  1. java.lang package:

    2. This package is automatically imported into every Java program and contains classes that are fundamental to the design of the Java language, such as Object, String, Math, and System.
                            import java.lang.*;
                        
                        public class MyClass {
                           public static void main(String[] args) {
                              String s = "Hello, world!";
                              System.out.println(s);
                           }
                        }
  2. java.util package:

    3. This package contains classes that are useful for performing a variety of utility operations, such as working with collections, handling dates and times, and generating random numbers.
                            import java.util.*;
                        
                        public class MyClass {
                            public static void main(String[] args) {
                                List list = new ArrayList<>();
                                list.add("apple");
                                list.add("banana");
                                list.add("orange");
                                System.out.println(list);
                            }
                        }
  3. java.io package:

    4. This package contains classes for performing input/output operations, such as reading and writing files and streams.
                            import java.io.*;
                        
                        public class MyClass {
                            public static void main(String[] args) {
                                try {
                                    FileInputStream fis = new FileInputStream("file.txt");
                                    int c;
                                    while ((c = fis.read()) != -1) {
                                        System.out.print((char) c);
                                    }
                                    fis.close();
                                } catch (IOException e) {
                                    e.printStackTrace();
                                }
                            }
                        }
  4. java.net package:

    5. This package contains classes for performing networking operations, such as connecting to remote hosts, sending and receiving data over sockets, and handling URLs.
                            import java.net.*;
                        public class MyClass {
                          public static void main(String[] args) {
                            try {
                              URL url = new URL("http://www.example.com/");
                              HttpURLConnection con = (HttpURLConnection) url.openConnection();
                              con.setRequestMethod("GET");
                              BufferedReader in = new BufferedReader(new InputStreamReader(con.getInputStream()));
                              String inputLine;
                              while ((inputLine = in.readLine()) != null) {
                                System.out.println(inputLine);
                              }
                              in.close();
                            } catch (IOException e) {
                              e.printStackTrace();
                            }
                          }
                        }
  5. java.sql package:

    6. This package contains classes for performing database operations, such as connecting to databases, executing SQL queries, and managing transactions.
    Example:

                                        import java.sql.*;
                                    
                                    public class MyClass {
                                      public static void main(String[] args) {
                                        try {
                                          Connection conn = DriverManager.getConnection("jdbc:mysql://localhost:3306/mydatabase", "root", "password");
                                          Statement stmt = conn.createStatement();
                                          ResultSet rs = stmt.executeQuery("SELECT * FROM customers");
                                          while (rs.next()) {
                                            System.out.println(rs.getString("name"));
                                          }
                                          rs.close();
                                          stmt.close();
                                          conn.close();
                                        } catch (SQLException e) {
                                          e.printStackTrace();
                                        }
                                      }
                                    }

User Defined Packages

In addition to the built-in packages in Java, you can also create your own packages in order to organize your code and make it more modular and reusable. User-defined packages in Java allow you to group related classes and interfaces together and provide a unique namespace for your code to prevent naming conflicts with other packages.


    Here's an example of how to create a user-defined package in Java:

  1. Create a new folder for your package in your project directory, and give it a meaningful name that reflects the purpose of your package. For example, you might create a package called "com.example.utilities" for a set of utility classes.


  2. Create your classes or interfaces inside this folder, and include a "package" statement at the top of each file to specify the name of your package.
    3. For example:

                            package com.example.utilities;
                        
                        public class StringUtils {
                            public static boolean isNullOrEmpty(String str) {
                                return str == null || str.isEmpty();
                            }
                        }
  3. In order to use your user-defined package in another part of your code, you need to import it using the package name.
    4. For example: 
                                    import com.example.utilities.StringUtils;

                                public class MyClass 
                                    {
                                public static void main(String[] args) 
                                        {
                                        String s = "Hello, world!";
                                        if (StringUtils.isNullOrEmpty(s))
                                            {
                                            System.out.println("The string is null or empty.");
                                            }
                                        else {
                                            System.out.println("The string is not null or empty.");
                                            }
                                        }
                                    }
    Note: that the package name must match the directory structure of your package, starting with the root directory of your project. In this example, the package name is "com.example.utilities", so the corresponding directory structure would be:

    project/
                                src/
                                    com/
                                        example/
                                                utilities/
                                                        StringUtils.java
                                      ...
    User-defined packages in Java provide a powerful way to organize your code and make it more modular and reusable. By following the conventions for package naming and structure, you can create a well-designed and maintainable codebase that is easy to understand and extend.
